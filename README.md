# Discord Music Bot

Discord Music Bot — це бот для відтворення музики у голосовому каналі Discord.
Також у проєкті є простий веб-інтерфейс, через який можна додавати треки в чергу, керувати відтворенням, дивитися поточний трек, прогрес і змінювати гучність.

## Можливості

* підключення бота до голосового каналу;
* відтворення музики з YouTube;
* додавання треків у чергу;
* команди `/join`, `/play`, `/skip`, `/pause`, `/resume`, `/leave`;
* веб-сторінка для керування музикою;
* відображення поточного треку;
* прогрес відтворення;
* регулювання гучності;
* робота з `.env` для зберігання токена.

## Опис основних файлів

### `bot.py`

Короткий файл для запуску проєкту.
Він запускає основну логіку з `main.py`.

### `main.py`

запуск проєкту

### `bot/client.py`

Створює клієнт Discord-бота, налаштовує intents і підключає команди.

### `bot/commands.py`

Містить команди Discord-бота:

* `/join`
* `/play`
* `/skip`
* `/pause`
* `/resume`
* `/leave`

Цей файл відповідає за взаємодію користувача з ботом через Discord.

### `bot/helpers.py`

Допоміжні функції для Discord-бота.
Наприклад: перевірка голосового каналу, підключення до voice channel, отримання voice client.

### `bot/search_view.py`

Файл для роботи з інтерактивним вибором треків, якщо використовується пошук через Discord-інтерфейс.

### `core/queue.py`

Логіка черги треків.
Тут зберігаються треки, які були додані користувачами.

### `core/events.py`

Події Discord-бота, наприклад `on_ready`.

### `core/cache.py`

Кеш для тимчасового зберігання даних, щоб не виконувати зайві запити повторно.

### `core/auth_proxy.py`

Допоміжний файл для роботи із зовнішніми сервісами або проксі-логікою.

### `services/audio.py`

Основна логіка відтворення аудіо.
Тут використовується FFmpeg, керування play/pause/resume/stop і гучністю.

### `services/progress.py`

Відповідає за прогрес відтворення треку: час, тривалість і позицію пісні.

### `services/spotify.py`

Допоміжна логіка для обробки Spotify-посилань або назв треків.
в цьому файлі зроблено так що коли ви кидаєте посилання на музику з спотіфай бот шукає її по назві в ютубі та вмикає його через ютуб, тому що спотіфай блокує пряме передавання музики в бота через захист від музикальних ботів

### `web/routes.py`

Flask routes для веб-інтерфейсу.
Через цей файл сайт отримує дані про трек, чергу, гучність і кнопки керування.

### `templates/index.html`

фронт енд сайту 

### `static/style.css`

CSS-стилі сайту: дизайн, кнопки, та інше

### `utils/logger.py`

Допоміжний файл для логування помилок і дій бота.

## Створення Discord-бота

1. Перейдіть на сайт Discord Developer Portal.
2. Створіть новий застосунок.
3. У розділі **Bot** створіть бота.
4. Скопіюйте токен бота.
5. У розділі **Privileged Gateway Intents** увімкніть:

```text
Message Content Intent
```

Також бажано увімкнути:

```text
Server Members Intent
Presence Intent
```

Але для базової роботи музичного бота головне — `Message Content Intent`.

## Права для бота

Під час створення invite-посилання потрібно вибрати такі scopes:

```text
bot
applications.commands
```

Для прав бота потрібно вибрати:

```text
View Channels
Send Messages
Read Message History
Use Slash Commands
Connect
Speak
Use Voice Activity
```

## Встановлення

Клонуйте репозиторій:

```bash
git clone https://github.com/removvi/-OP.git
cd -OP
```

Створіть віртуальне середовище:

```bash
python -m venv .venv
```

Активуйте його:

```bash
.venv\Scripts\activate
```

Встановіть залежності:

```bash
pip install -r requirements.txt
```

## Налаштування `.env`

Створіть файл `.env` у корені проєкту.

Приклад:

```env
DISCORD_TOKEN=your_discord_bot_token_here
```

Файл `.env` не потрібно завантажувати на GitHub, тому що там зберігається токен бота.

## Запуск

```bash
python bot.py
```

Після запуску бот має з’явитися онлайн у Discord.

Веб-інтерфейс відкривається у браузері за адресою:

```text
http://127.0.0.1:5000
```

## FFmpeg

Для роботи музики потрібен FFmpeg.

Якщо FFmpeg не встановлений глобально, потрібно завантажити його і додати шлях до `ffmpeg.exe` у системний PATH або вказати шлях у коді.

## Основна ідея архітектури

Спочатку вся логіка могла бути в одному великому файлі, але це незручно для підтримки.
Тому проєкт розділений на кілька частин:

* `bot/` — логіка Discord-бота;
* `web/` — логіка сайту;
* `services/` — музика, прогрес, Spotify;
* `core/` — черга, кеш, події;
* `utils/` — допоміжні функції.

Такий поділ робить код зрозумілішим і спрощує пошук помилок.
