# Курсова робота: Discord Music Bot з веб-панеллю та алгоритмічними модулями

## Опис проєкту

Цей проєкт є Discord-ботом для відтворення музики у голосових каналах. Бот підтримує slash-команди, чергу треків, керування відтворенням, гучністю та має веб-панель на Flask.

Додатково у проєкт інтегровані теми курсової роботи: генератори, ітератори, memoization, priority queue, async-функції, stream processing, EventEmitter, authentication proxy та logging decorator.

## Основні можливості

- підключення бота до голосового каналу;
- відтворення музики за назвою або посиланням;
- черга треків;
- пріоритетна черга;
- пауза, продовження, пропуск треку;
- веб-панель керування;
- відображення поточного треку та часу;
- регулювання гучності;
- логування дій;
- кешування результатів пошуку.

## Реалізовані теми курсової

### Task 1: Generators and Iterators

У проєкті реалізовано генератор `track_id_generator`, який створює нескінченну послідовність ID для треків.

Також реалізовано timeout iterator `consume_iterator_with_timeout`, який споживає значення з ітератора протягом заданого часу.

### Task 2: Project Setup

Проєкт має структуру Python-проєкту, README-файл, `.gitignore`, окремий файл `config.py` для токена, а також може бути розміщений на GitHub.

### Task 3: Memoization

Реалізовано клас `Memoizer`, який кешує результати пошуку треків. Це дозволяє не виконувати повторний запит до `yt-dlp`, якщо такий трек уже шукався раніше.

Підтримується:
- обмежений розмір кешу;
- TTL;
- LRU-логіка видалення старих елементів.

### Task 4: Bi-Directional Priority Queue

У проєкті реалізовано клас `BiDirectionalPriorityQueue`.

Він підтримує:
- додавання треків з пріоритетом;
- отримання найстарішого елемента;
- отримання найновішого елемента;
- отримання елемента з найвищим або найнижчим пріоритетом.

Ця структура використовується для черги музики.

### Task 5: Async Array Function Variants

Реалізовано функцію `async_map`, яка є асинхронним аналогом `map`.

Вона дозволяє паралельно обробляти список елементів через `asyncio.gather`.

### Task 6: Large Data Processing with Streams

Відтворення музики реалізовано через streaming. Бот не завантажує весь аудіофайл у пам’ять, а передає аудіопотік напряму через FFmpeg.

Це дозволяє працювати з великими аудіофайлами без повного завантаження в оперативну пам’ять.

### Task 7: Reactive Communication with EventEmitter

У проєкті реалізовано власний `EventEmitter`.

Він підтримує:
- підписку на події;
- відписку від подій;
- виклик усіх слухачів події.

Приклади подій:
- `track_added`;
- `track_started`.

### Task 8: Authentication Proxy

Реалізовано клас `AuthProxy`, який може додавати авторизаційні заголовки до HTTP-запитів.

Підтримуються типи авторизації:
- API Key;
- JWT;
- OAuth;
- без авторизації.

У проєкті цей механізм використовується для HTTP-запитів, пов’язаних зі Spotify oEmbed.

### Task 9: Logging Decorator

Реалізовано декоратор `@log(level="INFO")`.

Він підтримує:
- логування виклику функції;
- логування завершення функції;
- логування помилок;
- вимірювання часу виконання;
- підтримку sync та async функцій.
